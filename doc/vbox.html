WebDAV CGI Virtual Box

Getting Started

  1. Download WebDAV CGI Virtual Box
  2. Unpack (tar -avxf webdavcgi-virtualbox.tar.lzma or with 7-Zip),
    add machine WebDAV CGI (Ubuntu 12.04)/WebDAV CGI (Ubuntu 12.04).vdi to Virtual Box (Machine > Add...),
    and start WebDAV CGI Virtual Box (Virtual Box required)
  3. Call the web interface (please ignore SSL warnings and accept the certificate): https://localhost:4443/
    and login with user testuser1 and password testuser1

About the folders

  • home - is a virtual link from /data/home to /home/testuser1
  • projects - is a project folder with public share support, revision controlled files (RCS), and POSIC ACL support
  • scratch - is a temporary file storage
  • dbb - uses a SQLite database for all data (/tmp/webdav*db ; The database content is lost after reboot.)

About the Virtual Box

  • Virtual Box:
    • 1 CPU
    • 4 GB RAM
    • 2 harddrives: sda: root (ext4, quota) and swap, sdb: data (ext4) with POSIX ACL support and group quotas
    • Port forwarding: 4443 to 443 for https and 2222 to 22 for ssh
  • OS: Ubuntu 12.04 LTS 64bit:
    • Apache 2.2 with rewrite, ssl, and cgid modules
    • PostgreSQL database server for WebDAV locks and properties
    • LDAP server:
      • Base DN: dc=webdavcgi,dc=sf,dc=net
      • Admin password: adminuser
      • Entries: 4 users (testuser1, testuser2, public, quotatemplate) and 4 groups (testuser1, testuser2, public) in 2 organizational units
  • SSH access through port 2222: Login adminuser (password: adminuser); ssh -l adminuser localhost -p 2222
  • root access via adminuser: sudo -i
  • installation scripts: /root/install/
  • update scripts: /root/update.*
  • helper scripts (LDAP users/groups): /home/adminuser/scripts

Troubleshooting Guide

No connection to https://localhost:4443/

  • check WebDAV CGI Virtual Box machine is started: look into text console of the machine
  • sometimes the port 4443 is in use: look for a free port and change the port forwarding rules of the WebDAV CGI Virtual Box
